Инструкция по запуску приложения

1. Скачиваем проект:


2. Открываем его в IDEA.

3. Необходимо создать базу данных с именем: technique_shop

Host: localhost
Port: 5432
URL: jdbc:postgresql://localhost:5432/technique_shop

User и Password отсутствуют.

4. После создания БД и успешного подключения к ней, заходим в файл sqlForDB.sql и выполняем скрипты по созданию
и заполнению таблиц БД.

5. Запускаем приложение.

6. Далее можно работать с приложением через web браузер по url:

http://localhost:8080/ - переход на стартовую страницу

http://localhost:8080/find - выдает весь реестр техники

http://localhost:8080/find/name/{имя типа техники} - выдает список всех типов и моделей, выбранного типа техники
(можно передать одновременно разные типы техники через запятую без пробела)

http://localhost:8080/find-color?color={выбранные цвета} - выдает список всей техники данного цвета (можно передать
одновременно разные цвета техники через запятую без пробела)

http://localhost:8080/find-price?min={цена от}&max={цена до} - выдает список всей техники в выбранном диапазоне цен

http://localhost:8080/find-att?name={имя типа техники}&attname={название уникальной характеристики}&attvalue=значение
характеристики

http://localhost:8080/find-stock?name={имя типа техники}&stock={true или false} - выдает список всей техники по выбранному
типу, которая есть в наличии (можно передать одновременно разные типы техники через запятую без пробела)

http://localhost:8080/find-size?name={имя типа техники}&min={размер от}&max={размер до} - выдает список всей техники по
выбранному типу техники (можно передать одновременно разные типы техники через запятую без пробела) и заданному диапазону
размеров

http://localhost:8080/sort?name={имя типа техники}&sorting={тип сортировки: по возрастанию - asc, по убыванию - desc} -
выводит список всей техники по выбранному типу техники в алфавитном порядке (asc) или обратном порядке (desc)

http://localhost:8080/sort-price?name={имя типа техники}&sorting={тип сортировки: по возрастанию - asc, по убыванию - desc} -
выводит список всей техники по выбранному типу техники по возрастанию цены (asc) или по убыванию цены (desc)

7. Для добавления новой модели лучше использовать Postman.

    - добавляем новый Request через +
    - выбираем тип запроса POST
    - отправляем тело запроса вот в таком формате (значения можно ставить любые) :

    name =
    {
        "typeId": 5,
        "name": "testmodel",
        "serialNumber": 444444,
        "color": "Blue",
        "size": 10,
        "price": 1099990,
        "isInStock": true,
        "modelAttributeDto": [
          {
            "name": "Attr1",
            "value": "Value1"
          },
          {
            "name": "Attr2",
            "value": "Value2"
          },
          {
            "name": "Attr3",
            "value": "Value3"
          }
        ]
    }


    Расшифровка названия полей в теле запроса:

        typeId = id типа модели, в которую будем добавлять
        name = имя модели
        serialNumber = серийный номер модели
        color = цвет модели
        size = размер модели
        price = цена модели
        isInStock = наличие на складе (true / false)
        modelAttributeDto = уникальные характеристики в виде массива значение (name - имя, value - значение)

8. Также для проверки работоспособности прилоежния, после его запуска, можно перейти по ссылке:
http://localhost:8080/swag-run.html

Во всех методах можно вводить значения для проверки, результат будет выводиться на экран после нажатия на Execute.

Для проверки возможности добавления новой модели, надо удалить весь предложенный шаблон запроса и использовать шаблон
из п.7. Надо его скопировать и вписать туда любые валидные значения.
